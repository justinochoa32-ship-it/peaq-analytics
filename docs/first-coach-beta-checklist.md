# First Coach Beta Checklist

Use this as the repeatable flow for the first 5 coach demos.

## Before Inviting

- Confirm `https://app.peaqanalytics.com` loads the sign-in screen.
- Create a test coach account and confirm email/password reset works.
- Run one manual report, save it, refresh, and confirm it remains in the workspace.
- Import the demo CSV, save complete rows, and confirm one bulk-save message appears.
- Open one athlete profile and confirm report history, comparison, archive, and restore work.
- Print/save the PEAQ Profile PDF and confirm the one-page report is clean.
- Download a Story PNG from a saved report and confirm it saves as a vertical image.

## Invite Message

Hey Coach,

I am opening up a small beta group for PEAQ Analytics. It turns testing data into coach-ready athlete profiles, one-page reports, saved report history, and progress comparisons.

Here is the beta link: https://app.peaqanalytics.com

I will walk you through setup and the first import so you can test it with your own athletes.

## Onboarding Call Flow

1. Create the coach account.
2. Confirm the email if prompted.
3. Add one athlete manually and save a report.
4. Import a CSV with a few athletes.
5. Open the Athlete Library and filter/sort the roster.
6. Open an athlete profile and review score, archetype, limiter, strengths, and bucket scores.
7. Print/save the PEAQ Profile PDF.
8. Download a Story PNG if the coach wants a shareable visual.
9. Archive and restore a sample athlete so roster cleanup is clear.

## Feedback Questions

- Did the profile language make sense immediately?
- Did any score, limiter, or archetype feel confusing?
- Was CSV import clear enough to use without help?
- Was the one-page PDF useful for athletes or parents?
- Would the Story PNG be useful for social sharing?
- What part felt slow, unclear, or missing?

## Known Boundaries For Beta

- Scoring standards are fixed for this beta.
- There is no team-wide export yet.
- Story PNG is a simple vertical share card, not a full custom social editor.
- Coaches should use archive instead of delete for roster cleanup during beta.
